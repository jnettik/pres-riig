We need to understand a couple things before we talk responsive images

We've been oversimplfying images for a long time

Fill our designs with them.

Blob them into WYSIWYG editors.

Drupal can distance us from a lot the complexity.

Images are complex

At their simplest, images are small pockets of structured content.

  • src attribute
  • alt attribute
  • title attribute

Images are complex

Images come with a lot of unique properties.

  • Dimensions: height and width
  • File type: .jpg, .gif, .png, etc.
  • Compression, transparency, animation...
  • File size

Images are important

Images make up an increasing majority of every page on the internet.

  • 63% of pages are made of images.
  • 1.3Mb of the total page weight.
  • Use of images has doubled in the last three years.

Responsive images are more complex

Like every other part of what we do, responsive web design forced us to start thinking about these things, and so much more.

Responsive images are more complex

On top of the things mentioned above, we also need to consider...

  • different resolutions across screens.
  • fluid layouts and changing screen sizes.
    • changes in screen orientation as you change devices.
  • browser support for newer file types.
  • metered, expensive, unreliable mobile networks.

Why do we care?

Images on the web are a complex problem. We can't expect a simple solution.

Current solutions are not simple...

... however, Drupal does make things simpler.

img tag

This is required for every solution. Every other technique builds on to HTML's core img tag.

Responsive images progressively enhances the img tag.

srcset attribute

Attribute to inform the browser of alternate image sources.

Best used when trying to address resolution use cases.

srcset attribute

srcset comes in two flavors:

picture element

New element in HTML5 to define different image sources based on varying criteria.

srcset allows the browser to make educated decisions on source.

picture dictates to the browser which source we want.

picture element


Media condition: stripped down media queries.

srcset: defines the source to use at this condition.
